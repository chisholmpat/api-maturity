API Maturity Instructions

When to Use

Client wants to assess existing or new API Strategy and whether it is ready to pursue an APIs as a provider, consumer, business, or market.

How to Use

1. Each tab there is a series of questions that help determine where a customer is on the maturity curve for that enterprise dimension. First you will give the customers a series of API Characteric questions along with some deeper interview questions to help them think about their current API dynamics and the customer responses should be documented. You will rate the customers response on a scale of 0 to 5 gaining consensus on the score of course. This score is primarily from the customers perspective advised by you the assessor but it is used as a quantitative measurement.

  • 0 Don't do it
  • 1 Planned
  • 2 In progress
  • 3 Partially Implemented
  • 4 Complete (Mature)

2. Then you will give the customer a self assessment API Maturity for the enterprise dimension under discussion. This is the customers view only without any commentary or guideance other then to explain what it means to be mature in that API characteristic for that enterprise demension. The customer will rate themselves on a scale of 0 to 3 and it is their self-assessed rating.

  • 0 Novice
  • 1 Progressing
  • 2 Mature
  • 3 Partially Implemented
  • 4 Complete (Mature)
